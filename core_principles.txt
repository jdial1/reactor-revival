The Reactor Substrate: Core Design Principles
I. Mathematical Foundationalism (The Tick & The Pulse)

The simulation must reject "magic number" bloat in favor of formalized thermodynamic laws.

    Quadratic Divergence: The engine must strictly enforce the divergence between Power (Linear/Multiplicative scaling) and Heat (Quadratic scaling).

    The Pulse Priority: The "Neutron Pulse" is the primary variable of the substrate. Adjacency is not a bonus; it is the mathematical input for the generation phase.

    Integer Determinism: To prevent "Floating Point Flicker," all core resource calculations (Money, EP) are handled via Decimal libraries, but simulation-level physics (Heat/Pressure) are solved using high-performance TypedArrays in the Web Worker.

II. Topological Fluidity (The Thermal Bus)

The grid is not an array of objects; it is a mathematical topology defined by edges and nodes.

    The DAG Solver: Heat is treated as a fluid. The simulation resolves thermal transfer via a Directed Acyclic Graph (DAG). Instead of iterating through "Inlets" then "Exchangers," the system calculates "Thermal Pressure" (Heat/Containment) across the entire grid and moves energy along the gradient in a single batch.

    Abstract Adjacency: The engine uses a Compiled Adjacency Map. Orthogonal, Cross (Row/Col), and Radial interactions are pre-calculated upon grid-resize, allowing the physics tick to remain

    Functional Separation: Modern architecture must support the separation of the "Engine" (Cell Clusters) from the "Radiator" (Venting Arrays), enabling the transition from "Local Logic" (Surrounder builds) to "Global Logic" (Thermal Bus builds).

III. The Invariant Substrate (State over Action)

Progression is a derivation of state, not a sequence of mutations.

    Modifier Derivation: Upgrades are pure data. The system never imperatively changes a value (e.g., rows += 1). Instead, the Derived Substrate recalculates the active modifiers (Grid Dimensions, Transfer Rates, Global Buffs) every time the upgrade set changes.

    Transactional Persistence: Saving and Loading are governed by a strict Schema Pipeline. Every save is validated by Zod and transformed into a binary-packed representation, ensuring that the 50x50 grid remains lightweight and corruption-proof.

    The Command Queue: Any user interaction (Buying a part, selling power) is an "Intent" sent to the logic layer. This allows for native Undo/Redo and ensures the UI never directly mutates the simulation state.

IV. Deterministic Lifecycle (Autonomous Entropy)

The simulation is a self-governing state machine that exists independently of the view.

    The Accumulator: To solve the "Background Tab" problem, the engine uses a fixed-timestep accumulator. Real-world time is consumed to produce discrete simulation ticks, ensuring that 1 hour of offline time results in exactly 36,000 processed ticks.

    Worker Authority: The Web Worker is the "Source of Truth" for the simulation wallet. It handles component decay and "Auto-Buy" replacement internally, eliminating main-thread roundtrips during high-speed play.

V. Pure Reactive Projection (Stateless View)

The DOM is a "Read-Only" lens over the simulation state.

    Stateless Rendering: The UI is a pure function of the state. It can be destroyed and re-mounted at any time without losing data.

    Visual Delta Buffering: The Worker returns a compact Visual Delta Buffer (TypedArray) once per frame. The UI "projects" these pulses, explosions, and flows onto the canvas, decoupling high-frequency physics from the 60fps rendering limit.

    CSS-Driven Criticality: Thermodynamic constraints (Heat Ratio, Meltdown progress) are passed directly to CSS variables. Visual effects like screen jitter, color shifts, and heat haze are handled by the GPU via CSS/Shaders, freeing the JS thread for logic.

VI. Strategic Symmetry (The Min Constraint)

The ultimate goal is the architecting of perfect mathematical balance.

    The Bottleneck: Progression (Exotic Particles) is throttled by the lower of two values: Total Power Produced vs. Total Heat Dissipated.

    The Architect's Journey: The substrate forces the player to transition from a Manual Operator (Venting heat by hand) to a Systems Architect (Designing autonomous, balanced, and perpetual thermal loops).
