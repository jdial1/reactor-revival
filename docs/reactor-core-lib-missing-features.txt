================================================================================
reactor-core-lib — missing / incomplete features (host cutover leftovers)
================================================================================
Date: 2026-07-18 (reductive 1.0 roadmap — hardened binary exits)
Host: reactor-revival
Lib version in use: reactor-core-lib@1.2.12 (npm)
Import alias: "reactor-core"


PURPOSE
-------
Host is a UI wrapper + bridge + test suite. Simulation lives in reactor-core-lib.
Bridge is mandatory (inactive paths throw).

Authority cutover (physics ownership) is DONE. Remaining path to public 1.0 is
REDUCTIVE only: refine, delete, consolidate, and polish — do not add gameplay
systems, modes, parts, upgrades, or parallel architectures.

HARD LAWS (close loopholes — soft exits are forbidden):
  1. Unidirectional: UI → lib Command → bridge.dispatch → session → snapshot → UI.
     No production host→session stomps (preserveHostScalars defaults, etc.).
  2. Dumb data: host projection payloads are methodless POJOs. Display logic
     is pure functions in the UI layer — not class wrappers with helpers.
  3. Dumb bridge: dispatch(intent) ≈ session.execute(intent) — zero translation
     switch/case middleware. UI builds exact lib Command payloads. No facade zoo.
  4. No game mirror in Valtio: valtio holds UI ephemeral state only (tabs, modals,
     prefs). Money/heat/grid/levels render from bridge snapshot/observer — never
     mirrored into host state proxies.
  5. Harness isolation: sync helpers for tests must be physically unloadable
     from the production bundle.
  6. Soak is CI: S1–S8 are Playwright e2e specs — manual checkbox is not an exit.
  7. Binary exits: each Step exit is a pass/fail invariant, not "shrinking",
     "readable", or "clean enough".
  8. QW list FROZEN: no new Quick Wins; PRs must hit Trust Spine (G3–G6).

Doc authority for release intent: THIS FILE wins when other docs lag.
  design-foundations.md / architecture-rules.md / core_principles.txt still
  describe product identity, laws, and physics vocabulary — keep those.
  Their "worker kernel / dual-physics / mid-cutover" implementation maps are
  historical; do not plan 1.0 work from push-candidates or §6 paths that
  contradict DONE sections below.

PUBLIC 1.0 — WHAT IT SHOULD LOOK LIKE (definition of done)
================================================================================
Dated: 2026-07-18. This is the product + engineering picture of a tagged
v1.0.0 — not a feature wishlist. TEN PHASES (Steps 1–10) below are the
execution spine that gets here.

1.0 is a CLEAN SHIP of the game that already exists after authority cutover:

--------------------------------------------------------------------------------
A. Player-facing product shape (what "1.0" feels like)
--------------------------------------------------------------------------------
Identity (from design-foundations north star — still valid):
  Trustworthy browser IC2-style reactor sim: spatial adjacency puzzle,
  deterministic pulse physics, industrial operator fantasy. Theorycrafting
  and teachability live in-engine — not wiki/spreadsheet dependent.

In a 1.0 build the player can, end-to-end:

  LOOP
    Place / sell / vent / toggle / buy upgrades via one UI→intent path each.
    Tick runs; heat/power/meltdown/sell/spend match session authority.
    Heat overlays, tooltips, and control deck read projected session facts.

  PROGRESSION
    Chapter objectives teach mechanics; claim/auto-complete via session.
    Prestige / reboot / doctrine respec work through bridge; EP weave is
    honest and visible. Achievements are optional chrome, not a second econ.

  THEORYCRAFT
    Blueprint planner / compare / share codes already ship — they stay.
    Same session physics sample path; no parallel "planner math."

  PERSISTENCE & RETURN
    Save v3 + migrations load cleanly; .reactor export/import works.
    Welcome-Back / offline fast-forward is chunked, risk-aware, and does
    not clobber mid-flight economy. Meltdown recovers Recovered Blueprint
    before tile clear (presentation order correct).

  TRUST SURFACE
    Installable PWA; offline-capable enough for a session; What's New /
    changelog on version bump; leaderboard optional (not a ship blocker).
    Critical path usable on desktop + tablet/mobile without broken chrome.

PLAYER SOAK (gated — pass/fail before tag; desktop + one mobile viewport):
  HARD RULE: S1–S8 must be fully automated in e2e/*.spec.js via Playwright and
  run on CI. Manual verification is NOT a valid exit for these paths. Map each
  S-id 1:1 to a named e2e spec (extend engineering.spec.js / hazards.spec.js
  or add sibling specs). Step 9d / §E fail red if any S-path lacks CI e2e.

  S1 Fresh install → chapter teach through first prestige-relevant objective
  S2 Place / sell / vent / toggle / buy upgrade once each; deck matches session
  S3 Save → kill tab → reload → economy + grid + objective index intact
  S4 Welcome-Back FF → no money/EP clobber; heat risk does not soft-lock
  S5 Meltdown → Recovered Blueprint before tile clear; resume playable
  S6 Prestige / reboot / doctrine respec once each; EP weave consistent
  S7 .reactor export/import round-trip once
  S8 PWA install + offline enough for a short session

What 1.0 does NOT need to feel like:
  A content dump, a redesign, a cloud product, or a "complete genre checklist."
  Partial systems from design-foundations (undo history, deeper achievement
  catalog, meltdown-before-FF projection polish) are nice — not tag gates.
  CSS byte contests, knip zero-debt, and full harness rewrites are not tag
  gates unless listed as hard invariants below.

B. Engineering shape (what the codebase looks like at tag)
--------------------------------------------------------------------------------
Architecture (must hold — binary):

  session (reactor-core-lib)  = sole authority for tick, heat, money/EP,
                                upgrade levels, place/sell/vent, meltdown eval,
                                objective claim checks
  bridge                      = dispatch(libCommand) ≈ session.execute — tick +
                                project-out + hydrate-on-load; no translation zoo
  host projection             = methodless POJOs from getSnapshot()/project()
  host UI                     = pure formatters + lit templates rendering snapshot;
                                valtio = UI ephemeral only (never sim data)
  host services               = audio, PWA, leaderboard (no sim authority)
  tests                       = sim → createGameSession / lib state only;
                                host → projection/UI with mocks; soak → Playwright

Honesty bar at tag (binary):

  - package.json "1.0.0" matches real git tag v1.0.0 from green main
  - One version story (no Game.version disagreeing with package/tag)
  - Full `npm test` is sole merge/release gate; CI fails on lint *warnings*
  - Knip unused-export count does not grow above committed baseline
  - Zero production calls of harness sync APIs (physically excluded from bundle)
  - Zero production host→session runtime stomps (preserveHostScalars gone /
    default false and unused on hot paths; loadEconomyFromHost hydrate-only)
  - Zero sim fields (money/heat/grid/levels) in valtio proxies
  - bridge.dispatch is passthrough — no Intent→Command remap switch
  - Logging via core/logger.js only; ESLint bans console.* in public/src
  - Deploy: test → css → build:sw → PWA check (NO regex console-strip step)
  - Playwright e2e covers S1–S8 on CI (manual soak does not count)
  - Known issues frozen with severity; money/heat/save desync = tag blocker
  - README launch link and scripts match what ships

Reductive bar (not vanity LOC):

  - Dual systems collapsed to one way per job (scrap inventory + Step 3)
  - Domain class wrappers deleted or reduced to zero-method data + factories;
    display = pure UI functions (formatPartForDisplay, etc.)
  - Bridge public surface = attach + passthrough dispatch + tick/project/hydrate
  - Valtio for UI state only (tabs, modals, preferences); zero simulation data
    (grid, economy) mirrored in host state managers
  - Net-negative *dual paths* preferred over net-negative lines
  - CSS: orphan selectors deleted; payload size NOT a tag gate
  - QW list frozen — no hygiene theater PRs during trust spine

"Good enough" vs "perfect" at 1.0:

  KEEP imperfect but honest:
    Objective/achievement *display* chrome if single-path and non-authoritative
    Host meltdown presentation / recovered blueprint ordering
    Lib afford-flag / CLAIM_OBJECTIVE gaps via lib Command + dispatch only
    main.css / objectives.css large if layouts work (no automate-to-shrink)

  NOT good enough for tag:
    Authority regressions; save/load broken; dual meltdown/econ evaluators
    Anemic domain classes with methods pretending to be DTOs
    Bespoke bridge methods OR switch(intent.type) translation middleware
    Valtio (or any host store) mirroring money/heat/grid/levels
    Harness sync importable from production modules
    Manual-only soak checkboxes without Playwright CI coverage
    New Quick Wins while G3–G6 open
    Feature freeze ignored; CI green via deleted tests / skipped lint
    package.json version pretending the tag already shipped
    Soft exits ("readable", "shrinking", "clean enough") marked DONE
